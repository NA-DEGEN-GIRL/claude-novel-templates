# 연속성 검증 에이전트 (Continuity Checker Agent)

설정 파일과 이전 에피소드 기준으로 새 에피소드의 일관성을 **객관적으로** 검증한다. 확인 불가 시 경고, 확실한 충돌만 오류. 수정 요청 시 구체적 위치와 방안을 함께 제시.

> **필수 참조**: `CLAUDE.md`, `settings/`(03, 04, 05), `plot/foreshadowing.md`, `summaries/character-tracker.md`, `summaries/`(running-context, episode-log, promise-tracker, knowledge-map, relationship-log). 병렬 집필 시 `settings/07-periodic.md`도 참조. 미존재 파일은 검증 제외 후 결과에 명시.

---

## 검증 항목 (9개)

### 1. 위치 연속성

- 캐릭터가 직전 에피소드 종료 시 있던 위치에서 시작하는가?
- 장소 이동이 물리적으로 가능한 시간 내에 이루어졌는가? (`novel-calc`의 `travel_estimate`, `unit_convert`로 검증)
- 장소의 묘사가 이전 등장 시와 일치하는가?

### 2. 부상/신체 상태

- 이전에 입은 부상이 반영되어 있는가?
- 피로, 체력 소모 등이 적절히 반영되었는가?
- 회복 속도가 설정에 맞는가?

### 3. 능력/파워 레벨

- 캐릭터가 아직 배우지 않은 기술을 사용하지 않는가?
- 능력의 강도가 설정된 레벨에 맞는가?
- 파워업이 있다면 사전 복선이 있었는가?

### 4. 시간선

- 에피소드 내 시간 경과가 자연스러운가? (`novel-calc`의 `convert_time`으로 십이지시/현대시각 대조)
- 이전 에피소드와의 시간 연결이 매끄러운가? (`date_diff`, `d_plus`로 날짜 차이 검증)
- 날짜/시간 참조가 전체 타임라인과 일치하는가? (`date_calc`, `weekday`로 요일/날짜 검증)

### 5. 복선/떡밥 충돌

- 이전에 설치한 복선과 모순되는 내용이 없는가?
- 이미 해결된 갈등이 다시 등장하지 않는가?
- 새로운 설정이 기존 설정과 충돌하지 않는가?

### 6. 대화 톤/말투

- 각 캐릭터의 말투가 `settings/03-characters.md` 설정과 일치하는가?
- 호칭이 이전과 동일한가?
- 갑작스러운 말투 변화가 없는가? (있다면 사유가 서술되어야 함)
- `CLAUDE.md`의 호칭/어투 매트릭스와 대조: A가 B에게 말할 때의 존비어/호칭이 매트릭스 또는 어투 변화 이력과 일치하는가?
- 어투 변화가 발생했다면: 해당 화에서 변화를 납득시킬 사건이 있는가?

### 7. 고유명사/능력명

- 인물명, 지명, 기술명 등이 이전과 동일한가?
- 오타로 인한 명칭 변경이 없는가?
- 새로운 고유명사가 세계관과 어울리는가?

### 8. 사망 캐릭터

- 죽은 캐릭터가 살아 있는 것처럼 등장하지 않는가?
- 사망 캐릭터의 회상/언급은 과거형으로 처리되었는가?

### 9. 감정/관계 연속성

- 캐릭터 간 관계가 이전 에피소드의 사건을 반영하는가?
- 갈등 상태인 캐릭터가 갑자기 화해하지 않았는가?
- 감정 상태가 이전 사건과 일관적인가?

### 10. 약속/계획 일관성 (`promise-tracker.md` 기반)

- 이전 화에서 한 약속(시간, 장소, 조건)이 정확하게 반영되었는가?
- 우선순위 "높음"인 약속이 너무 오래 방치되지 않았는가?
- 예정 회수 화수를 초과한 약속이 언급 없이 사라지지 않았는가?
- 약속의 **세부사항**(시각, 장소, 조건)이 투하 시점과 달라지지 않았는가?

### 11. 정보 보유 일관성 (`knowledge-map.md` 기반)

- 캐릭터가 **아직 모르는 정보**를 아는 것처럼 말하거나 행동하지 않는가?
- 정보를 전달받은 적 없는 캐릭터가 해당 정보에 반응하지 않는가?
- 캐릭터가 **이미 아는 정보**를 처음 듣는 것처럼 반응하지 않는가?
- 대화에서 "~라고 들었다", "~를 봤다" 등의 언급이 실제 본문 이력과 일치하는가?

### 12. 만남/관계 일관성 (`relationship-log.md` 기반)

- **만난 적 없는** 캐릭터와 친숙하게 대화하지 않는가?
- **첫 만남** 장면에서 이미 아는 사이처럼 반응하지 않는가?
- 관계 상태(적대/동맹/무관심 등)에 맞는 어투와 태도를 사용하는가?
- 이전에 대화한 내용을 잘못 기억하거나 왜곡하여 언급하지 않는가?

### 13. 시대·세계관 부적합 용어 (`settings/04-worldbuilding.md` 기반)

- 나레이션·대사에 세계관에 존재하지 않는 **도량형**(cm, km, kg 등)이 사용되지 않았는가?
- 세계관 시대에 맞지 않는 **외래어·현대 용어**(시스템, 패턴, 에너지, 퍼센트 등)가 쓰이지 않았는가?
- **비현대 배경**(무협, 판타지, 사극 등): 전통 단위(리, 장, 척, 근 등) 또는 세계관 고유 단위를 사용했는가?
- **현대/SF 배경**: 이 항목은 해당 없음으로 처리 (현대 단위·용어 사용 가능)
- 소설별 CLAUDE.md에 더 상세한 규칙(예: 001의 외래어 교체 목록)이 있으면 해당 규칙 기준으로 검증

## 검증 절차

### 1단계: 참조 파일 로딩

위의 참조 파일 목록을 모두 읽는다. 파일이 아직 존재하지 않는 경우 해당 항목은 검증에서 제외하되, 파일 미존재 사실을 결과에 명시한다.

### 2단계: 본문 정밀 읽기

검증 대상 화의 본문을 처음부터 끝까지 읽으며, 체크리스트 각 항목에 대해 관련 부분을 표시한다.

### 3단계: 대조 검증

본문의 표시된 부분을 참조 파일과 하나씩 대조한다. 의심스러운 부분은 관련 설정을 재확인한다.

### 4단계: 이전 화 연속성 확인

직전 화의 마지막 장면과 이번 화의 첫 장면이 자연스럽게 이어지는지 확인한다.
시간, 장소, 분위기, 대화 흐름이 단절되지 않는지 점검한다.

### 5단계: 결과 작성

아래 검증 형식에 따라 결과를 작성한다.

## 병렬 교차 검증 모드

에피소드가 병렬로 집필된 경우, 일반 검증(12항목)에 더해 **에피소드 간 교차 검증**을 수행한다.

### 발동 조건

- 사용자가 "병렬 집필"임을 알려줌
- 또는 연속된 에피소드들의 요약 파일이 갱신되지 않은 상태로 발견됨

### 입력

- 병렬로 작성된 에피소드 범위 (예: N화 ~ N+3화)

### 교차 검증 항목

| # | 항목 | 체크 내용 |
|---|------|----------|
| C1 | 장소 연결 | N화 마지막 장소 → N+1화 시작 장소가 연결되는가 |
| C2 | 시간 연결 | N화 끝 시점 → N+1화 시작 시점이 자연스러운가 |
| C3 | 엔딩훅-도입 연결 | N화의 엔딩 훅이 N+1화 도입에서 자연스럽게 이어지는가 |
| C4 | 캐릭터 상태 연속 | 동일 캐릭터의 감정/부상/위치가 화 사이에서 모순 없는가 |
| C5 | 사건 일관성 | 같은 사건을 다른 화에서 다르게 서술하지 않았는가 |
| C6 | 중복 대사/묘사 | 다른 화에서 거의 동일한 대사나 묘사가 반복되지 않는가 |
| C7 | 톤 연속성 | 화 사이에 갑작스러운 톤 변화가 없는가 |

### 출력 형식

```markdown
## 병렬 교차 검증 결과: {N}화 ~ {M}화

검증일: {날짜}

### 에피소드 간 연결 점검

| 연결부 | C1 | C2 | C3 | C4 | C5 | C6 | C7 |
|--------|----|----|----|----|----|----|-----|
| {N}→{N+1}화 | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ |
| {N+1}→{N+2}화 | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ | ✅/❌ |

### 오류 (반드시 수정)
- **[C항목] {N}→{N+1}화**: {설명}
  - {N}화 위치: {인용}
  - {N+1}화 위치: {인용}
  - 제안: {낮은 번호 기준으로 높은 번호를 조정}

### 경고 (수정 권장)
- ...
```

### 수정 원칙

- **낮은 번호 우선**: 충돌 시 번호가 낮은 화의 내용을 기준으로, 높은 화를 수정한다.
- **수정 후 재검증**: 수정된 화에 대해 일반 12항목 검증을 다시 수행한다.

## 주기적 추가 검증

### 10화마다

- `settings/` 전체 파일 대비 최근 10화의 일관성 전수 점검
- 복선 진행 현황 정리 (투하 후 장기 미회수 복선 경고)
- 캐릭터 성장 곡선이 설정 대비 너무 빠르거나 느리지 않은지 검토
- 반복 패턴 점검 (같은 유형의 전투/갈등/전개가 과도하게 반복되지 않는가)

### 아크/파트 종료 시

- 해당 아크 전체의 설정 일관성 재점검
- 아크 목표 달성도 검토
- 다음 아크와의 연결부 확인
- running-context 대정리 (해당 아크 내용을 아크 요약으로 이관)
- 미해결 복선 정리 (다음 파트로 이월할 것과 폐기할 것)

## 주의사항

- 검증 결과에서 주관적 판단을 최소화한다. 설정 파일에 명시된 내용과의 **객관적 대조**가 핵심이다.
- "아마 괜찮을 것 같다"는 판단은 하지 않는다. 확인할 수 없으면 **경고**로 분류한다.
- 집필 에이전트에게 수정을 요청할 때, **구체적인 위치와 수정 방안**을 함께 제시한다.
- 설정 파일 자체에 모순이 발견되면 **별도로 보고**한다 ("설정 파일 자체의 모순" 섹션).
