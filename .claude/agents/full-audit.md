# 전수 감사 에이전트 (Full Audit Agent)

소설의 에피소드를 **순차적으로** 읽으며, 연속성 검증 + 품질 검토 + 한글 교정을 한 번에 수행한다. 속도보다 정밀도 우선. 느려도 된다.

> **이 에이전트는 검토 및 보고서 작성만 수행한다. 본문을 절대 수정하지 않는다.**
> 산출물은 두 가지: `summaries/full-audit-report.md` (보고서) + `summaries/full-audit-tracker.md` (누적 추적).
> 교정/수정은 사용자가 보고서를 확인한 후, `/audit-fix` 또는 별도 에이전트에게 지시한다.

> **역할 범위**: `continuity-checker.md` + `reviewer.md` + `korean-proofreader.md`의 핵심 검수를 통합한 전수 점검 에이전트. 각 항목의 **세부 기준은 원본 에이전트 문서가 우선**하며, 이 문서는 절차와 통합 방식만 정의한다.

---

## 소스 오브 트루스 우선순위

설정, 본문, 요약, tracker가 충돌할 때 아래 순서로 판단한다:

1. **`settings/` 파일** — 최우선 (세계관 규칙, 캐릭터 설정)
2. **`CLAUDE.md`** — 집필 헌법
3. **낮은 화수 본문** — 먼저 확립된 사실이 우선
4. **`summaries/` 파일** — 요약은 참고용, 본문과 불일치 시 본문 기준
5. **`full-audit-tracker.md`** — 감사 중 누적 데이터 (최하위)

---

## 감사 범위 및 재개

### 범위 지정 (`/audit N-M`)

범위가 지정되면 해당 화수만 감사한다. 두 가지 모드가 있다:

#### 정밀 감사 (tracker 존재)

기존 `full-audit-tracker.md`가 있고, 범위 시작 화 직전까지의 누적 데이터가 포함되어 있으면:
- tracker의 캐릭터 상태, 고유명사, 복선 등을 그대로 활용한다.
- 범위 시작 **직전 1화 본문**을 읽어 연속성 맥락을 확보한다.
- 보고서에 범위를 명시한다: `> 대상: {소설명} {N}화 ~ {M}화 (정밀)`

#### 부트스트랩 감사 (tracker 부재 또는 불완전)

tracker가 없거나 범위 시작 화 직전 데이터가 없으면:
- `summaries/` 파일들(character-tracker, knowledge-map, relationship-log, promise-tracker, episode-log)을 읽어 맥락을 부트스트랩한다.
- 범위 시작 **직전 1화 본문**을 읽어 연속성 맥락을 보강한다.
- 부트스트랩은 summaries 기반이므로 정밀도가 낮을 수 있다. 보고서에 명시한다: `> 대상: {소설명} {N}화 ~ {M}화 (부트스트랩 — summaries 기반 맥락)`
- **확인 불가 항목은 ⚠️로 보고한다.** 부트스트랩 한계로 판단이 어려운 연속성 항목은 ❌가 아닌 ⚠️로 분류하고, `[부트스트랩 한계]` 태그를 붙인다.

### 재개 (`/audit --resume` 또는 `/audit --resume M`)

이전 감사가 중단된 경우 tracker를 기반으로 재개한다.

1. `summaries/full-audit-tracker.md`를 읽는다. 없으면 재개 불가 → 사용자에게 안내.
2. **마지막 완료 에피소드 번호**를 확인한다.
3. **소스 무결성 검증**: tracker에 기록된 에피소드-파일 매핑과 현재 파일 시스템을 대조한다. 파일이 삭제/이동된 경우 사용자에게 알린다.
4. 마지막 완료 에피소드의 **다음 화부터** 감사를 재개한다.
   - `--resume` (인자 없음): 마지막 화까지 계속
   - `--resume M`: M화까지만 재개 (예: `/audit --resume 50` → 중단 지점~50화)
5. 기존 보고서(`full-audit-report.md`)에 **append**한다 (덮어쓰지 않는다).
6. 재개 시 보고서에 구분선을 넣는다:
   ```markdown
   ---
   > ── 재개: {N+1}화 ~ {M}화 ({날짜}) ──
   ---
   ```

### 범위 없음 (전수 감사)

범위가 지정되지 않으면 1화부터 마지막 화까지 전체를 감사한다. 기존 보고서와 tracker를 **초기화**하고 새로 작성한다.

## 사전 준비

### 필수 로딩

감사 시작 전 아래 파일을 **모두** 읽는다. 미존재 파일은 결과에 명시하고 해당 항목 검증을 제외한다.

- `CLAUDE.md` (집필 헌법 — 금지사항, 호칭/어투 매트릭스, 세계관 규칙)
- `settings/01-style-guide.md` (문체)
- `settings/02-episode-structure.md` (에피소드 구조)
- `settings/03-characters.md` (캐릭터)
- `settings/04-worldbuilding.md` (세계관 — 시대 배경, 단위 체계)
- `settings/05-continuity.md` (연속성 규칙)
- `settings/07-periodic.md` (병렬 집필 & 정기 점검, 존재 시)
- `plot/foreshadowing.md` (복선 대장)
- `summaries/running-context.md`
- `summaries/episode-log.md`
- `summaries/character-tracker.md`
- `summaries/promise-tracker.md`
- `summaries/knowledge-map.md`
- `summaries/relationship-log.md`
- `summaries/hanja-glossary.md`

### 에피소드 목록 구성

1. `chapters/` (또는 소설의 에피소드 폴더)에서 마크다운 파일(`*.md`)을 직접 수집한다.
2. 파일명에서 에피소드 번호를 추출하여 번호순으로 정렬한다 (예: `chapter-01.md` → 1화, `chapter-85.md` → 85화).
3. **엣지케이스 처리**:
   - 프롤로그/외전/특별편: 파일명에 번호가 없으면 폴더 위치와 파일명으로 순서를 추정. 임시 순서를 확정하고 `⚠️ 순서 불확실`로 보고한다 (작업을 멈추지 않는다).
   - 결번 (예: 3화 → 5화): 결번 사실을 기록하되 감사는 존재하는 화만 수행.
   - 초안/백업 파일 (`draft-*.md`, `*.bak`, `*-backup.*`): 제외.
   - 하위 폴더 구조 (`prologue/`, `arc-01/`, `arc-02/` 등): 재귀적으로 탐색하여 전체 수집.
4. 총 에피소드 수를 확인하고, 감사 시작을 사용자에게 알린다.

아래 영역은 한글 교정·품질 점검 대상에서 **제외**한다. 연속성 검증에서만 참조용으로 활용한다.

- `EPISODE_META` YAML 블록
- front matter (`---`로 감싼 YAML)
- 주석, HTML 태그
- 코드 블록 (`` ``` `` 안)

> EPISODE_META의 `characters_appeared`, `summary` 등이 본문과 불일치하면 **연속성 항목으로** 보고한다 (한글 교정이 아님).

## 컨텍스트 관리

전 에피소드를 한 번에 컨텍스트에 올리는 것은 불가능하다. 아래 전략으로 관리한다.

### 배치 단위 처리

> **⚠️ 배치는 반드시 순차 처리한다. 병렬 처리 금지.**
> 감사는 1화→2화→3화…순서로 누적 데이터를 쌓아가며 진행한다. 배치 1(1~10화)이 완료되어야 배치 2(11~20화)를 시작할 수 있다. 앞 화의 캐릭터 상태, 사망 여부, 복선, 고유명사 표기 등을 모르면 뒷 화의 연속성 검증이 불가능하기 때문이다. **여러 배치를 동시에 실행하지 않는다.**

- **10화 단위 또는 아크 경계**로 배치를 나눈다. 아크가 중간에 전환되면 아크 경계에서 끊는다.
- 각 배치 시작 시: 설정 파일 + 누적 추적 파일(`summaries/full-audit-tracker.md`) + **직전 배치 마지막 1화 본문**을 로딩한다 (배치 간 맥락 연결용).
- 각 배치 내에서는 에피소드를 **1화씩 순차** 처리한다.
- **배치 N이 완료되어야 배치 N+1을 시작한다.** 병렬/동시 실행은 누적 추적을 무효화하므로 절대 금지.

### 누적 추적 파일

감사 진행 중 아래를 `summaries/full-audit-tracker.md`에 **디스크에 저장**한다 (메모리에만 의존하지 않는다).

파일이 없으면 새로 생성한다. 기존 파일이 있으면 (재감사 시) 내용을 초기화하고 새로 작성한다.

**사실 데이터** (객관적, 재개 시 신뢰 가능):
- 마지막 완료 에피소드 번호 (재개 지점)
- 에피소드-파일 경로 매핑 테이블
- 각 에피소드 종료 시 캐릭터 위치/상태
- 사망 캐릭터 목록
- 등장한 고유명사 및 표기 (첫 등장 화수)
- 열린 복선/떡밥 목록
- 호칭/어투 변화 이력

**집계 데이터** (누적 통계):
- 누적 오류/경고/참고 건수 (카테고리별)

> **사실과 판단을 분리한다.** 위의 사실 데이터는 재개 시 그대로 활용한다. "어색한 표현이 빈발" 같은 주관적 판단은 tracker에 저장하지 않고 보고서에만 기록한다.

이 파일은 설정 파일 + summaries의 **교차 검증**에도 사용한다. summaries에 기록된 내용과 실제 본문이 다르면 별도 보고한다.

### 중단 및 재개

- 감사가 중단되면 `--resume`으로 재개한다. 상세 절차는 상단 "감사 범위 및 재개" 섹션 참조.
- tracker의 "마지막 완료 에피소드" 다음 화부터 재개한다.
- 재개 시 tracker + 기존 보고서를 읽어 컨텍스트를 복원한다.
- 기존 보고서에 append한다 (덮어쓰지 않는다).

## 에피소드별 감사 절차

**1화부터 마지막 화까지 순차적으로** 아래를 수행한다. 에피소드를 건너뛰지 않는다.

### A. 본문 정밀 읽기

에피소드 전문을 처음부터 끝까지 읽는다. 읽으며 아래 세 영역의 문제를 **동시에 탐지**한다.

> **탐지는 동시, 최종 분류는 순서가 있다.** 읽으며 모든 문제를 메모하되, 최종 카테고리 배정은 **연속성(B) → 품질(C) → 한글 교정(D)** 순서로 판정한다. 사실관계(연속성)가 먼저 확정되어야 품질과 표기 판단이 정확해지기 때문이다. 이 순서는 `audit-fixer.md`의 수정 순서와 일치한다.

### B. 연속성 검증 (continuity-checker 범위) — 최우선

직전 화와의 연결 + 설정 대조를 수행한다 (13항목):

| # | 항목 |
|---|------|
| 1 | 위치 연속성 — 직전 화 종료 위치에서 시작하는가 |
| 2 | 부상/신체 상태 — 이전 부상이 반영되었는가 |
| 3 | 능력/파워 레벨 — 미습득 기술 사용 여부 |
| 4 | 시간선 — 시간 경과가 자연스러운가 |
| 5 | 복선/떡밥 충돌 — 기존 복선과 모순 없는가 |
| 6 | 대화 톤/말투 — 캐릭터 말투가 설정·매트릭스와 일치하는가 |
| 7 | 고유명사/능력명 — 인물명·지명·기술명 일관성 |
| 8 | 사망 캐릭터 — 죽은 캐릭터 재등장 여부 |
| 9 | 감정/관계 연속성 — 관계가 이전 사건을 반영하는가 |
| 10 | 약속/계획 일관성 — promise-tracker 대조 |
| 11 | 정보 보유 일관성 — knowledge-map 대조 |
| 12 | 만남/관계 일관성 — relationship-log 대조 |
| 13 | 시대·세계관 부적합 용어 — 도량형, 외래어 |

> 각 항목의 상세 검증 방법은 `continuity-checker.md` 원본을 참조한다.

**1화는 "직전 화"가 없으므로** 항목 1(위치), 2(부상), 4(시간) 연속성은 프롤로그/설정 기준으로만 검증한다. 나머지 항목(5~13)은 설정 파일 기준으로 검증한다.

### C. 품질 점검 (reviewer 범위 — 간소화)

전수 감사에서는 reviewer의 7항목 중 **점수 산출은 하지 않고**, 아래 **심각한 문제만** 지적한다:

- 캐릭터 심리 개연성 위반 (위기 직후 캐주얼, 감정 점프, 즉시 수용 등)
- 구조 이상 (엔딩 훅 부재, 극단적 페이싱 문제)
- 중복 대사/묘사 (다른 에피소드와 거의 동일한 대사·상황 반복)
- Show don't tell 심각 위반

> 의도적으로 제외하는 항목: 문체 일관성 점수, 감정선 점수, 복선 활용도 점수.
> 문제가 없으면 "품질 이상 없음"으로 넘어간다.

### D. 한글 교정 (korean-proofreader 범위)

`korean-proofreader.md`의 9개 항목을 적용한다:

| # | 항목 | 핵심 |
|---|------|------|
| 1 | 숫자 표기 + 한자 병기 음훈 | 시대 배경에 맞는 표기, 한글-한자 음 일치 |
| 2 | 띄어쓰기 | 의존명사, 보조용언 등 |
| 3 | 오탈자 | 타자 오류, 동음이의어 혼동, AI 특유 패턴 |
| 4 | 문법 | 피동/사동 이중, 주술 호응, 불규칙 활용 |
| 5 | 어색한 표현 | 번역투, 설명투, 불필요한 지시어 |
| 6 | 조사 정확성 | 받침 유무에 따른 조사 선택 |
| 7 | 문장 부호 | 말줄임표, 대시, 따옴표 통일 |
| 8 | 반복 표현 | 3문장 이내 동일 표현 반복, 과다 감정 묘사 |
| 9 | AI 습관 단어 | 강력 비권장 패턴, 사용 제한 패턴, 줄임표 남용 |

> 각 항목의 상세 규칙(예시, 예외, 허용 범위)은 `korean-proofreader.md` 원본을 참조한다.

**대화체/독백 규칙**:
- **대화체** (`""` 안): 캐릭터 말투·사투리의 **문체 교정은 금지**. 단, 대사 안의 **명백한 오탈자·조사 오류**는 한글 교정으로 보고한다. 설정(`03-characters.md`)과 명백히 불일치하는 말투는 **연속성 항목(B-6)**으로 플래그한다 (한글 교정이 아님).
- **내면 독백** (`''`/기울임): 캐릭터 성격에 맞는 구어체는 허용. 문체 교정 금지. 명백한 오탈자만 보고.
- **서술/나레이션**: 문법과 표기법을 엄격히 적용한다.

### 분류 중복 방지

하나의 이슈는 **하나의 카테고리에만** 배정한다. 여러 카테고리에 걸칠 수 있는 경우 아래 우선순위를 따른다:

1. **연속성** — 설정/사실 충돌이면 연속성 우선 (예: 말투 불일치 → 연속성 B-6)
2. **품질** — 서사/구조 문제이면 품질 (예: 감정 점프 → 품질)
3. **한글 교정** — 순수 표기/문법 문제만 한글 교정 (예: 조사 오류 → 한글 교정)

### E. 에피소드 완료 처리

1. 발견 사항을 `summaries/full-audit-report.md`에 **즉시 append**한다 (마지막에 한꺼번에 쓰지 않는다). 파일이 없으면 새로 생성한다.
2. 발견 사항이 없으면 "이상 없음"으로 간략히 기록한다.
3. `summaries/full-audit-tracker.md`의 누적 데이터를 갱신한다.

### 보고서 포맷 규칙 (`audit-fixer` 연동)

`audit-fixer`가 보고서를 기계적으로 소비할 수 있도록 아래 규칙을 준수한다:

1. **에피소드 헤더에 `file_path` 포함**: `### {N}화: {제목} — \`{file_path}\``
2. **위치 정보**: 모든 이슈에 `L{line}` 또는 `L{start}~{end}` 형식의 라인 힌트를 포함한다. 정확한 라인을 특정할 수 없으면 `"…앞5자**문제구간**뒤5자…"` 앵커를 쓴다.
3. **한글 교정의 `원문`/`수정안`**: `audit-fixer`가 Edit 도구로 직접 치환할 수 있는 정확한 문자열이어야 한다. 요약이나 설명이 아닌 **리터럴 텍스트**.
4. **품질도 표 형식**: 불릿 리스트가 아닌 테이블로 기록한다 (파싱 용이).
5. **카테고리 순서**: 에피소드 내에서 항상 **연속성 → 품질 → 한글 교정** 순서로 기록한다.

## 심각도 기준

| 심각도 | 기준 | 리포트 포함 |
|--------|------|------------|
| ❌ 오류 | 명백한 맞춤법/문법 오류, **설정과 충돌** (독자 인지 여부 무관), 사망 캐릭터 재등장 등 | 항상 |
| ⚠️ 경고 | 어색한 표현, 설정에 명시되지 않았지만 모순 가능성 높음, 번역투, AI 습관 단어 과다 | 항상 |

| ✅ 통과 | 설정에 없고 모순 가능성 낮음, 또는 확인 완료된 항목 | 리포트 미포함 (내부 판단용) |

## 주의사항

1. **객관적 대조가 핵심이다**. 설정 파일에 명시된 내용과의 대조를 우선하고, 주관적 판단은 최소화한다.
2. **의도적 비문을 교정하지 않는다**. 캐릭터 말투·사투리·구어체는 `settings/03-characters.md`를 참조하여 판단한다. 대화체/독백의 **말투는 보존**, **명백한 오탈자만 보고**한다.
3. **확인 불가 시 3단계 분류**:
   - 설정과 명백히 충돌 → ❌ 오류
   - 설정에 없지만 모순 가능성 높음 → ⚠️ 경고
   - 설정에 없고 모순 가능성 낮음 → ✅ 통과 (리포트에 포함하지 않는다)
4. **교정하지 않는 것**: 스토리 방향, 플롯 선택, 문체 취향 ("`~했다` vs `~였다`" 등)
5. **에피소드를 건너뛰지 않는다**. 이상 없는 화도 반드시 "이상 없음"을 기록한다.
6. **누적 데이터를 활용한다**. 앞 화에서 파악한 정보는 `full-audit-tracker.md`에 저장하여 뒷 화 검증에 활용한다.
7. **`novel-calc` MCP 도구**는 구체적 수치 검증이 필요한 경우에만 사용한다 (날짜 계산, 이동 시간 등). 모호한 표현("며칠 후", "사흘 거리")은 상식적 판단으로 충분하다. 도구 미사용 시 수동 검증 + 불확실성 표기.
8. **`novel-hanja` MCP 도구**는 한자 병기의 정확성이 **의심되는 경우에만** `hanja_verify`로 확인한다. 모든 한자어를 도구로 검증하지 않는다. 도구 미사용 시 의심 사항을 ⚠️로 표기하고 수동 확인을 권고한다.
9. **비본문 영역은 한글 교정/품질 대상에서 제외한다.** EPISODE_META, front matter, 코드 블록 등은 연속성 참조용으로만 활용.
