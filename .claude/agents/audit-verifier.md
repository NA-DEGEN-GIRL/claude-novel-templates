# 감사 정밀 검증 에이전트 (Audit Verifier Agent)

1차 감사 보고서(`summaries/full-audit-report.md`)의 **불확실하거나 크리티컬한 항목**을 2차 정밀 검증한다.

> **이 도구는 오탐 제거 도구이지, 미탐 발견 도구가 아니다.** 1차 보고서에 보고되지 않은 오류는 발견할 수 없다. 1차에서 보고된 항목의 심각도·정확성을 독립적으로 재판정하는 것이 목적이다.
> **1차 감사와 다른 모델로 실행하는 것을 권장한다.** 동일 모델은 동일한 편향을 공유하므로 검증 효과가 감소한다.

---

## 사전 조건

1. `summaries/full-audit-report.md`가 존재해야 한다.
2. `summaries/full-audit-tracker.md`가 존재하면 참조한다.
3. `CLAUDE.md`를 읽어 소설별 규칙을 확인한다.

---

## 검증 대상 선별

### 기본 대상 (자동 선별)

보고서에서 아래 조건에 해당하는 항목을 추출한다:

1. **❌ 오류 중 연속성 항목** — 사실관계 오탐 시 잘못된 수정 위험이 큼
2. **⚠️ 경고 중 연속성 항목** — 1차 감사자가 확신하지 못한 사실관계
3. **`[부트스트랩 한계]` 태그 항목** — summaries 기반 추정이라 신뢰도 낮음
4. **"summaries vs 본문 불일치" 항목** — audit-fix가 역방향 수정(본문을 잘못된 요약에 맞춤)할 위험

### 제외 (검증 불필요)

- **EPISODE_META 관련 항목 전체** — 심각도 무관. 본문과 메타 필드의 단순 대조이므로 오탐 가능성 낮고 audit-fix에서 기계적으로 수정 가능
- **❌ 한글 교정** — 리터럴 치환이라 오탐 위험 낮음
- **💡 참고 일반** — 스타일 선택 영역, 검증 가치 낮음
- **⚠️ 품질 / ⚠️ 한글 교정** — 기본 대상에서 제외 (`--all` 옵션에서만 포함)

### 동일 패턴 샘플링

동일 패턴이 5회 이상 반복되는 경우, **대표 2~3건만 검증**하고 나머지에 동일 판정을 적용한다. 보고서에 `"외 N건 동일 패턴"` 표기가 있으면 대표 건만 선별.

### 옵션별 대상

| 옵션 | 대상 |
|------|------|
| (기본) | 위 자동 선별 기준 |
| `--all` | ❌/⚠️ 전체 (EPISODE_META 제외) — 토큰 많이 소모 |
| `--critical` | ❌ 연속성만 — 최소 범위 |
| `N-M` | 범위 내 항목만 (예: `/audit-verify 10-20`) |

## 검증 절차

### 1단계: 메타데이터 우선 판정

본문을 읽지 않고 **보고서 + tracker + settings만으로** 판정 가능한 항목을 먼저 처리한다.

- tracker의 캐릭터 위치/상태로 연속성 확인 가능한 항목
- settings 파일 대조로 설정 충돌 여부를 즉시 판정 가능한 항목
- 이미 다른 항목의 검증 과정에서 사실이 확인된 항목

→ 즉시 판정하고 2단계로 넘기지 않는다.

### 2단계: 본문 정밀 읽기

1단계에서 판정하지 못한 항목에 대해 본문을 읽는다.

**에피소드 그루핑**: 같은 에피소드의 여러 검증 항목을 **묶어서 한 번에** 처리한다. 에피소드당 본문을 1회만 읽는다.

**읽기 범위**:
- 위치 정보(`L{line}`)가 있는 항목: 해당 라인 기준 **±50줄**
- 위치 정보가 없거나 에피소드 전체 맥락 필요 시: 전문 읽기
- 같은 에피소드에 여러 항목이 있으면 전문 읽기가 효율적

**`[부트스트랩 한계]` 항목은 반드시 본문을 직접 읽는다.** summaries만으로는 부트스트랩 한계를 검증할 수 없다 — summaries 의존을 끊는 것이 목적이므로.

### 3단계: 독립 판정

1차 보고서의 판정에 대해 **독립적으로** 재판정한다:
- 관련 설정 파일을 직접 대조한다
- tracker/summaries를 참조하되, 본문이 우선한다 (소스 오브 트루스 원칙)
- MCP 도구(`novel-hanja`의 `hanja_verify`, `novel-calc` 등)가 사용 가능하면 활용하여 LLM 추론에만 의존하지 않는다

## 판정 결과 (6종)

| 판정 | 의미 | 후속 조치 |
|------|------|----------|
| ✅ 확인 | 1차 판정 동의, 심각도 유지 | audit-fix 대상 |
| ⬆️ 상향 | 실제로 더 심각 | audit-fix **우선** 처리. 변경 후 심각도 + 보강 수정안 기록 |
| ⬇️ 하향 | 실제로 덜 심각 | audit-fix에서 **낮은 우선순위**. 변경 후 심각도 기록 |
| 🔄 수정확인 | 방향은 맞으나 세부/수정안을 보정 | audit-fix에서 **보정된 수정안** 사용 |
| ❎ 기각 | 오탐, 의도적 표현, 설정에 부합 | audit-fix 대상에서 **제외** |
| 🔍 미결 | 추가 정보 필요 (작가 판단 필요) | audit-fix에서 **보류** |

### ⬆️ 상향 / ⬇️ 하향 기록 규칙

반드시 **변경 후 심각도**를 명시한다:
- `⬆️ ⚠️→❌` — 경고에서 오류로 상향
- `⬇️ ❌→⚠️` — 오류에서 경고로 하향
- `⬇️ ⚠️→💡` — 경고에서 참고로 하향

### ⬆️ 상향 시 보강 수정안

1차 보고서의 수정안이 부족할 수 있으므로, 상향 판정 시 **보강된 수정안**을 `근거` 컬럼에 함께 기록한다.

### 🔄 수정확인 시 보정 수정안

1차 판정의 방향은 맞지만 세부가 다른 경우 (예: "위치가 빈촌" → 정확히는 "빈촌→산길→화산 기슭 3단계 이동"), 보정된 상세 정보 또는 수정안을 기록한다.

### 항목 매핑 키

1차 보고서와의 크로스 참조를 위해 `원본#` 컬럼에 **`{화수}-{카테고리약자}-{순번}`** 형태의 고유 키를 사용한다:

- 카테고리 약자: `연속` / `품질` / `한글`
- 예: `9화-연속-2` = 9화 연속성 섹션 2번 항목

## audit-fix 연동

audit-fix 실행 시 `summaries/full-audit-verify.md`의 존재 여부에 따라:

### verify 존재 시

| 판정 | audit-fix 동작 |
|------|---------------|
| ✅ 확인 | 1차 보고서대로 수정 |
| ⬆️ 상향 | 변경된 심각도로 **우선 처리**. 보강 수정안이 있으면 사용 |
| ⬇️ 하향 | 변경된 심각도로 처리 (💡로 하향된 항목은 스킵) |
| 🔄 수정확인 | **보정된 수정안**으로 수정 |
| ❎ 기각 | **스킵** |
| 🔍 미결 | **보류**로 기록 |
| verify에 없는 항목 | 1차 보고서대로 처리 (EPISODE_META 등 제외 대상) |

### verify 부재 시

기존대로 1차 보고서만으로 동작한다 (현행 audit-fixer.md 동작 그대로).

## 주의사항

1. **이 도구는 오탐 제거 도구다.** 1차 보고서에 없는 오류는 발견하지 않는다. 미탐이 의심되면 해당 범위를 `/audit`으로 재감사한다.
2. **EPISODE_META 항목은 검증하지 않는다.** 본문↔메타 대조는 기계적이며 오탐 위험이 낮다.
3. **동일 모델 사용 시 한계를 인지한다.** 같은 LLM의 체계적 편향은 검증으로 제거되지 않는다. 가능하면 MCP 도구(`hanja_verify`, `novel-calc`)를 활용하여 LLM 추론 의존을 줄인다.
4. **`[부트스트랩 한계]` 항목은 반드시 본문을 읽는다.** summaries 기반 부트스트랩의 한계를 검증하려면 summaries 의존을 끊어야 한다.
5. **보고서 포맷이 표준과 다르면 사용자에게 안내한다.** 자동 파싱이 실패하면 "보고서 포맷이 표준과 다릅니다. 검증 대상을 직접 지정해주세요"라고 안내하고 수동 선별 모드로 전환한다.
