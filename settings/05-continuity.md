# 연속성 관리 시스템

> 에피소드 간 일관성을 유지하기 위한 시스템.

---

## 1. EPISODE_META

매 에피소드 마지막에 아래 형식으로 메타데이터를 작성한다.

```yaml
episode: 1
title: "에피소드 제목"
summary: "에피소드 핵심 내용 2~4문장 요약"
date: "2026-02-25"
pov: "시점 캐릭터"
location: "주요 장소"
timeline: "작품 내 시간 위치"
characters_appeared:
  - 캐릭터1
  - 캐릭터2
key_events:
  - "주요 사건 1"
  - "주요 사건 2"
character_states:
  캐릭터1:
    physical: "상태 (부상, 피로 등)"
    emotional: "감정 상태"
    location: "현재 위치"
  캐릭터2:
    physical: "상태"
    emotional: "감정 상태"
    location: "현재 위치"
new_info:
  - "새로 밝혀진 정보"
foreshadowing:
  planted:
    - "새로 설치한 복선"
  recalled:
    - "회수한 복선 (원래 에피소드 번호)"
unresolved:
  - "미해결 떡밥 1"
  - "미해결 떡밥 2"
next_episode_hook: "다음 화 예고/암시"
```

### 작성 규칙

1. **모든 등장인물 기록**: 이름이 나온 모든 캐릭터를 `characters_appeared`에 포함
2. **상태 변화 추적**: 부상, 감정 변화, 위치 이동을 `character_states`에 기록
3. **복선 관리**: 새 복선은 `planted`, 회수한 복선은 `recalled`에 기록
4. **미해결 목록 유지**: 이전 에피소드의 미해결 사항 중 해결된 것은 제거, 새 것은 추가

## 2. 에피소드 요약 (summaries/)

매 에피소드 작성 후 `summaries/` 폴더에 요약을 작성한다.

### 요약 형식

```markdown
# EP.{{NUMBER}} - {{TITLE}} 요약

## 주요 사건
- (사건 1)
- (사건 2)

## 캐릭터 변화
- **{{CHARACTER}}**: (변화 내용)

## 새로운 정보/설정
- (새 정보)

## 미해결 사항
- (떡밥/갈등)

## 다음 화 연결
- (다음 화로 이어지는 요소)
```

### 요약 규칙

1. **분량**: 원고의 1/5 이하로 요약
2. **객관적 기술**: 감상이 아닌 사실만 기록
3. **스포일러 포함**: 요약은 내부 참고용이므로 모든 내용을 포함

## 3. 연속성 체크리스트

새 에피소드 작성 전 확인할 사항:

| # | 항목 | 확인 방법 |
|---|------|-----------|
| 1 | 직전 에피소드의 마지막 장면과 자연스럽게 이어지는가 | 직전 에피소드 마지막 200자 확인 |
| 2 | 캐릭터 위치가 물리적으로 타당한가 | EPISODE_META의 location 확인 |
| 3 | 부상/상태가 반영되었는가 | character_states 확인 |
| 4 | 능력/레벨이 설정과 일치하는가 | characters.md 확인 |
| 5 | 시간 흐름이 자연스러운가 | timeline 확인 |
| 6 | 캐릭터 말투가 일관적인가 | characters.md의 말투 확인 |
| 7 | 고유명사가 정확한가 | worldbuilding.md 확인 |
| 8 | 사망 캐릭터가 재등장하지 않는가 | characters.md 확인 |
| 9 | 미해결 떡밥과 충돌하지 않는가 | unresolved 목록 확인 |

## 4. 장기 연속성 관리

### 10화마다 정리

10화 단위로 아래를 업데이트한다:

1. **캐릭터 마스터시트** (`03-characters.md`): 변화된 관계, 새 캐릭터 추가
2. **세계관 설정** (`04-worldbuilding.md`): 새로 밝혀진 설정, 시간선 업데이트
3. **미해결 떡밥 목록**: 전체 떡밥 정리, 장기 미회수 떡밥 체크

### 파트/아크 전환 시

- 이전 파트의 전체 요약을 작성
- 캐릭터 상태를 리셋 포인트로 기록
- 미해결 떡밥 중 다음 파트로 이월할 것과 폐기할 것을 정리
