# 병렬 집필 & 정기 점검

> 병렬 집필 후 정합성 점검과 5화마다의 정기 점검 규칙을 정의한다.

---

## 1. 병렬 집필 후 정합성 점검

에피소드를 병렬로 집필한 경우, **집필 완료 후** 아래 절차를 반드시 수행한다.

### 병렬 집필 판단 기준

아래 중 하나라도 해당하면 "병렬 집필"로 간주:
- 여러 에이전트가 동시에 다른 화를 집필
- 요약 파일이 갱신되지 않은 화가 연속 존재
- 사용자가 명시적으로 병렬 집필임을 알려줌

### 각 에이전트 규칙

- 3.1~3.3은 그대로 수행 (맥락 로딩, 집필, 자체 검토)
- 3.4 연속성 검증: 자기 담당 범위 내에서만 수행
- 3.5 후처리에서 **요약 갱신은 보류** (충돌 방지)
- EPISODE_META 삽입은 수행

### 완료 후 정합 절차

1. **교차 연속성 검증**: `continuity-checker`의 "병렬 교차 검증" 모드로 에피소드 간 연결 점검
2. **충돌 수정**: 낮은 번호 기준으로 높은 번호를 조정
3. **요약 일괄 갱신**: 번호순으로 모든 요약 파일을 일괄 갱신

---

## 2. 정기 점검 (5화마다)

> 에피소드 단위 파이프라인으로 잡히지 않는 **누적 드리프트**를 방지한다.

### 트리거

- 직전 정기 점검 이후 5화 이상 집필 시
- 아크 전환 시점 (5화 미만이어도 수행)

### 점검 항목

| # | 항목 | 방법 |
|---|------|------|
| P1 | 요약 정합성 | `summary-validator` **일괄 감사(모드 B)** — 직전 점검 이후 작성된 에피소드 대상 |
| P2 | 복선 회수 시한 | `plot/foreshadowing.md`에서 회수 예정 시점이 지난 복선이 있는지 확인 |
| P3 | 캐릭터 상태 최신성 | `summaries/character-tracker.md`의 현재 상태가 **최신 에피소드 종료 시점**과 일치하는지 확인 |
| P4 | 미이행 약속 | `summaries/promise-tracker.md`에서 시한이 지났거나 방치된 약속이 있는지 확인 |
| P5 | running-context | `summaries/running-context.md`가 200줄 이내인지, 최신 상태를 정확히 반영하는지 확인 |
| P6 | 다중 소스 일괄 리뷰 | `gemini-feedback` **배치 모드(모드 B)**로 직전 점검 이후 에피소드를 일괄 리뷰. **성격 드리프트 확인 포함** — 주요 캐릭터 대사/행동이 `settings/03-characters.md`와 부합하는지 함께 점검 |
| P7 | 한글 품질 점검 | P6에서 텍스트 수정이 발생한 에피소드만 `korean-proofreader` 기준으로 재검수 |
| P8 | 메타 참조 금지 | 본문에서 "X화에서", "프롤로그에서", "1부에서" 등 에피소드 번호/구조명 참조 전수 검사 |

### 점검 후 조치

1. 발견된 이슈를 수정한다.
2. `running-context.md`를 최신 상태로 갱신한다.
3. 점검 결과를 커밋에 포함: `{소설명} {N}~{M}화 정기 점검 완료`
